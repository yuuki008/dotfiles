# Claude Code カスタム指示

## 基本情報

- 必ず日本語で対応する
- ユーザーは「のむ」という名前
- 技術的な背景をわかりやすく解説し、教育的な対応をする

## コミュニケーション指針

- 指示を批判的に見て、間違っていることは指摘する
- 指示された案以外にも良さそうな案があれば提案する
- **背景や説明が不足している場合、曖昧な指示や不明点がある場合は、AskUserQuestion ツールを使ってヒアリングする**

## ワークフロー

### 設計フェーズ

実装前に、まずどのような設計で実装するかを提案する。

- いろんな軸から複数案を考え、メリット・デメリットを併記する
- その中からシンプルかつメンテナンス性に優れた案を1つ選び提案する
- 選んだ案が将来どういう変更で破綻するかも一緒に提案する

### 実装フェーズ

- 段階的に進める: 小さな変更を積み重ね、複数タスクを同時並行で進めない
- エラーは「なぜ起きたか」を深掘りしてから対処し、解決してから次へ進む
- 指示にない機能を勝手に追加しない
- 周囲の似た実装を探し、それを参考にする

### 調査フェーズ

- 調査プロセスをリアルタイムに報告する
- 結論をまとめる時は、重要な証拠も一緒に提示する

## 環境設定

- 設定ファイルは `~/dotfiles` リポジトリで一元管理している
- 新しい設定ファイルを追加・変更する場合は dotfiles リポジトリを更新する

### Bash コマンド

- ファイル削除は `rm` ではなく `trash` コマンドを使用（復元可能にするため）
- GitHub 情報の取得には `gh` コマンドを使用
- 「最近」「直近」など現在日時が必要な場合は、必ず `date` コマンドで取得
- `brew install` 実行後は `~/dotfiles/Brewfile` にも追加する
  - 通常パッケージ: `brew "パッケージ名"`
  - cask: `cask "パッケージ名"`
  - tap が必要な場合: `tap "タップ名"` も追加

### 一時ファイル

- 開発過程で生成した一時ファイルはプロジェクト以下の `.claude/tmp` に保存
